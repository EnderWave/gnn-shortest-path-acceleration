# 仓库协议

本协议用于约束本项目后续文件组织、命名和实验记录。新文件进入仓库前，先按这里检查。

## 1. 文件命名

1. 项目内新增的源码、脚本、文档、图表和实验结果文件，默认使用中文命名。
2. 文件名要能直接看出内容，不使用含糊缩写，例如优先使用 `生成波尔图起终点热力图.py`，不要使用 `run.py`、`test.py`、`tmp.csv`。
3. 英文缩写只有在已经是通用术语时才保留，例如 `OD`、`OSM`、`DIMACS`、`CSV`、`PBF`。
4. 第三方原始数据文件、公开数据集原始文件、压缩包内部文件名，优先保留原名，避免影响引用、校验和复现。
5. 目录名可以暂时沿用已有结构；后续如果要统一目录名，需要单独迁移，不和实验代码修改混在一起。

## 2. 数据文件

1. 原始下载文件放在 `data/compressed/` 或 `data/raw/`，不直接改内容。
2. 清洗、抽样、吸附、统计后的文件放在 `data/processed/`。
3. 图片和可视化结果放在 `data/plots/`。
4. 大文件不轻易解压进仓库；能流式读取压缩包时，优先流式读取。
5. 每个新数据来源都要配一份中文说明，写清楚来源、许可、文件含义和后续处理方式。

## 3. 代码文件

1. 脚本要有明确入口，能从项目根目录直接运行。
2. 脚本参数要有默认值，默认值对应当前项目已有数据。
3. 不把一次性调试代码混进正式脚本。
4. 不在代码里写死个人机器路径。
5. 生成文件前要确保父目录存在。

## 4. 实验记录

1. 每次生成重要结果图或结果表，都要能追溯到脚本、输入数据和关键参数。
2. 基线结果、热力图、吸附成功率、查询耗时等结果文件，文件名要包含数据集和规模。
3. 临时探索可以放在本地，但进入仓库的结果必须可解释、可复现。

## 5. 当前例外

本仓库已经存在一些英文命名的历史文件、第三方数据文件和通用目录名。短期内不强行一次性重命名，避免破坏已有路径。新增自写文件先执行中文命名规则，历史文件后续按模块逐步整理。
