# 阶段三：参数与预算扫描

## 阶段状态

实验脚本已完成，全量实验尚未运行。

## 阶段目标

通过控制变量实验分析区域数量、区域大小和随机种子对索引规模、回退率与在线性能的
影响，并确定后续传统策略和 GNN 必须共同遵守的公平预算。

## 已完成工作

- 实现可断点续跑的扫描脚本。
- 支持已有配置自动跳过和 `--restart` 全量重跑。
- 支持控制变量扫描与完整参数网格两种模式。
- 每组实验记录压缩图规模、内部节点数、shortcut 数、回退率、预处理耗时、在线耗时、
  展开节点数、查询加速比例和正确率。
- 随机策略配置 5 个随机种子，用于统计均值和波动。

## 计划实验

控制变量模式共 42 组：

- 固定区域大小 512，扫描区域数量 50、100、200、400。
- 固定请求区域数 200，扫描区域大小 128、256、512、1024。
- 随机区域每个配置运行种子 1、2、3、4、5。
- OD 热点区域每个配置运行一次。
- 每组使用全部 98,082 条 OD，并进行基线/压缩配对验证。

## 当前缺失

仓库中尚不存在 `results/parameter_scan/porto_parameter_scan.csv`，因此目前不能回答：

- 收益是否随区域数量或大小稳定变化。
- 随机策略的均值、方差和置信程度。
- 热点策略是否在某些预算区间优于随机策略。
- 应以 shortcut 数、内部节点数还是压缩图规模作为主要预算。
- GNN 最终比较应采用哪组参数。

## 执行方式

先检查计划配置：

```bash
python scripts/run_parameter_scan.py --dry-run
```

再运行全量扫描：

```bash
python scripts/run_parameter_scan.py
```

脚本入口：[`scripts/run_parameter_scan.py`](../scripts/run_parameter_scan.py)。

## 完成标准

- 42 组配置全部写入扫描结果 CSV。
- 每组正确率均为 100%，否则先排查实现错误。
- 汇总随机种子的均值、标准差和最差结果。
- 绘制性能—预算、回退率—预算和预处理成本—预算关系。
- 明确推荐参数与最终公平预算，并更新本汇报的阶段结论。

## 下一阶段入口条件

只有本阶段完成并确定预算口径后，阶段四才能固定候选区域规模、模型输出和同预算选择
规则，避免 GNN 与传统策略在不同存储成本下进行不公平比较。
